# Start publishing with Markdown

This blog now runs as a static site that is built from Markdown files and deployed with GitHub Pages.

## Why this workflow works well

- No server is required.
- Content lives in your Git repository.
- Every push can trigger a fresh deployment.
- The published site is easier to host for free.

## Publishing a new post

1. Create a file inside `content/posts`.
2. Add the front matter fields.
3. Write the body in Markdown.
4. Commit and push the change.

## Optional comments

The post template can embed Giscus comments. Keep the feature disabled until you finish the GitHub Discussions setup in `content/site.json`.
